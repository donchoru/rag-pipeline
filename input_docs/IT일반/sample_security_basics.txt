정보보안 기초와 실무 적용

정보보안은 기밀성(Confidentiality), 무결성(Integrity), 가용성(Availability)의 CIA 삼각형을 기반으로 한다. 기밀성은 인가된 사용자만 정보에 접근할 수 있도록 하는 것이고, 무결성은 정보가 비인가된 변경 없이 정확하게 유지되는 것이며, 가용성은 필요할 때 정보에 접근할 수 있는 것이다.

인증과 인가

인증(Authentication)은 사용자가 누구인지 확인하는 과정이다. 비밀번호 기반 인증이 가장 기본적이지만, 비밀번호만으로는 보안이 충분하지 않다. 다중 인증(MFA, Multi-Factor Authentication)은 알고 있는 것(비밀번호), 가지고 있는 것(OTP 기기), 본인 자체(생체 인식)의 요소를 조합하여 보안을 강화한다.

OAuth 2.0은 서드파티 애플리케이션에 제한된 접근 권한을 부여하는 프로토콜이다. Authorization Code Grant, Client Credentials Grant, PKCE 등의 플로우가 있으며, 용도에 따라 적절한 플로우를 선택해야 한다. OpenID Connect(OIDC)는 OAuth 2.0 위에 인증 레이어를 추가한 프로토콜로, ID Token을 통해 사용자 정보를 확인할 수 있다.

JWT(JSON Web Token)는 클레임 기반의 토큰 형식으로, 서명을 통해 위변조를 방지한다. Access Token은 짧은 만료 시간(15분~1시간)으로 설정하고, Refresh Token으로 새로운 Access Token을 발급받는 패턴이 일반적이다. 토큰 저장은 httpOnly 쿠키를 사용하는 것이 XSS 공격에 안전하다.

인가(Authorization)는 인증된 사용자가 어떤 리소스에 접근할 수 있는지 결정하는 과정이다. RBAC(Role-Based Access Control)은 역할 기반으로 권한을 관리하고, ABAC(Attribute-Based Access Control)은 사용자, 리소스, 환경 등의 속성을 기반으로 더 세밀한 접근 제어를 수행한다.

웹 보안 취약점

OWASP Top 10은 웹 애플리케이션의 가장 치명적인 보안 위험 목록이다.

SQL Injection은 사용자 입력을 SQL 쿼리에 직접 포함하면 발생한다. 공격자가 입력값에 SQL 구문을 삽입하여 데이터를 유출하거나 삭제할 수 있다. 대응 방법은 파라미터화된 쿼리(Prepared Statement)를 사용하고, ORM을 활용하며, 입력값을 철저히 검증하는 것이다.

XSS(Cross-Site Scripting)는 악성 스크립트를 웹 페이지에 삽입하는 공격이다. Stored XSS는 서버에 저장되어 다른 사용자에게 전달되고, Reflected XSS는 URL 파라미터를 통해 즉시 반영된다. 대응 방법은 출력 시 HTML 이스케이프를 적용하고, Content-Security-Policy(CSP) 헤더를 설정하며, 입력값을 화이트리스트 방식으로 검증하는 것이다.

CSRF(Cross-Site Request Forgery)는 인증된 사용자의 권한으로 의도하지 않은 요청을 보내는 공격이다. 대응 방법은 CSRF 토큰을 사용하고, SameSite 쿠키 속성을 설정하며, Referer/Origin 헤더를 검증하는 것이다.

암호화

대칭 암호화는 암호화와 복호화에 같은 키를 사용한다. AES-256은 가장 널리 사용되는 대칭 암호화 알고리즘으로, 데이터 저장 시 암호화(Encryption at Rest)에 주로 사용된다. 키 관리가 핵심 과제이며, AWS KMS, Google Cloud KMS, HashiCorp Vault 등의 도구가 키를 안전하게 관리한다.

비대칭 암호화는 공개키와 개인키 쌍을 사용한다. RSA, ECDSA 등의 알고리즘이 있으며, TLS/SSL 핸드셰이크, 전자 서명, SSH 인증 등에 사용된다. HTTPS는 TLS 프로토콜을 사용하여 클라이언트와 서버 간 통신을 암호화한다. Let's Encrypt를 통해 무료 SSL 인증서를 발급받을 수 있다.

해시 함수는 임의 길이의 데이터를 고정 길이로 변환한다. 비밀번호 저장에는 bcrypt, scrypt, Argon2 같은 느린 해시 함수를 사용해야 한다. MD5와 SHA-1은 더 이상 안전하지 않으며, 최소 SHA-256 이상을 사용해야 한다. 솔트(salt)를 추가하여 레인보우 테이블 공격을 방어한다.

보안 운영

DevSecOps는 개발 프로세스 전체에 보안을 통합하는 접근법이다. SAST(Static Application Security Testing)는 소스 코드를 분석하여 취약점을 찾고, DAST(Dynamic Application Security Testing)는 실행 중인 애플리케이션을 테스트한다. SCA(Software Composition Analysis)는 오픈소스 라이브러리의 알려진 취약점을 검사한다.

침투 테스트(Penetration Testing)는 실제 공격자의 관점에서 시스템의 취약점을 찾는 활동이다. Burp Suite, OWASP ZAP 등의 도구를 사용하고, PTES(Penetration Testing Execution Standard) 등의 방법론을 따른다. 정기적인 침투 테스트와 버그 바운티 프로그램이 보안 수준을 높이는 데 효과적이다.

인시던트 대응은 보안 사고 발생 시의 체계적인 대응 절차이다. 탐지, 분석, 격리, 복구, 사후 분석의 단계로 진행된다. SIEM(Security Information and Event Management) 도구가 로그를 수집하고 이상 행위를 탐지한다. Splunk, Elastic Security, Microsoft Sentinel 등이 대표적이다.
