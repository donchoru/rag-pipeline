클라우드 인프라 구축 가이드

클라우드 컴퓨팅은 인터넷을 통해 컴퓨팅 리소스를 제공하는 모델이다. AWS, GCP, Azure가 3대 클라우드 서비스 제공자이다.

IaaS vs PaaS vs SaaS
IaaS(Infrastructure as a Service)는 가상 서버, 스토리지, 네트워크 등 인프라를 제공한다. EC2, Compute Engine이 대표적이다. 사용자가 OS부터 직접 관리해야 한다.

PaaS(Platform as a Service)는 애플리케이션 개발 플랫폼을 제공한다. App Engine, Cloud Run, AWS Elastic Beanstalk 등이 있다. 인프라 관리 부담 없이 코드에 집중할 수 있다.

SaaS(Software as a Service)는 완성된 소프트웨어를 서비스로 제공한다. Gmail, Slack, Notion 등이 SaaS이다.

컨테이너와 오케스트레이션
Docker는 애플리케이션을 컨테이너로 패키징하는 기술이다. Dockerfile로 이미지를 빌드하고, 어디서든 동일하게 실행할 수 있다.

Kubernetes(K8s)는 컨테이너 오케스트레이션 플랫폼이다. Pod, Service, Deployment, Ingress 등의 리소스로 컨테이너를 관리한다. 자동 스케일링, 롤링 업데이트, 자가 복구 기능을 제공한다.

CI/CD 파이프라인
CI(Continuous Integration)는 코드 변경을 자동으로 빌드하고 테스트한다. CD(Continuous Deployment)는 테스트를 통과한 코드를 자동으로 배포한다. GitHub Actions, Jenkins, GitLab CI가 널리 사용된다.

IaC(Infrastructure as Code)는 인프라를 코드로 관리하는 방식이다. Terraform, Pulumi, AWS CDK 등의 도구가 있다.
